# 云中书 / YunType — 开发分期计划

> 本文档定义开发里程碑、每期交付内容和验收标准。

---

## 总体规划

| 阶段 | 周期 | 核心交付 | 对应功能 |
|------|------|---------|---------|
| **Phase 1: MVP** | 2周 | 公众号排版核心流程 | F1+F2+F5+F7+F8 |
| **Phase 2: AI** | 1周 | AI分析+微调面板 | F3+F4+F6 |
| **Phase 3: 小红书** | 2周 | 小红书图片组生成 | F9（含字体系统） |

## Phase 1: MVP（公众号排版核心）

**目标**: 粘贴文章 → 随机推荐排版 → 预览 → 复制富文本到微信

### 交付清单

- [ ] 项目脚手架（Vite + React + TypeScript）
- [ ] 11套配色方案数据（colors.ts）
- [ ] 5种排版结构数据（layouts.ts）
- [ ] 4种装饰风格数据 + HTML模板（decorations.ts）
- [ ] 3种字体气质数据（typography.ts）
- [ ] 原子组合引擎（atoms/index.ts）
- [ ] Markdown 解析器（render/markdown.ts）
- [ ] 公众号 HTML 渲染器（render/wechat.ts）— 全部内联CSS，微信兼容
- [ ] 文章输入组件（ArticleInput）
- [ ] 实时预览组件（WechatPreview）— 375px 模拟
- [ ] "换一个"按钮 — 随机切换原子组合
- [ ] 复制富文本功能（export/clipboard.ts）
- [ ] 导出 HTML 功能（export/download.ts）
- [ ] 底部状态栏（字数、当前方案名称）
- [ ] Vercel 部署

### 验收标准

1. 粘贴一篇 Markdown 文章，点击"换一个"切换不同排版风格
2. 预览区实时显示排版效果
3. "复制富文本"后粘贴到微信公众号编辑器，样式完整保留
4. 660种组合中随机抽样10种，全部渲染正确

## Phase 2: AI 分析 + 微调

**目标**: AI 自动推荐排版 + 滑条微调

### 交付清单

- [ ] API 配置弹窗（ApiConfig 组件）
- [ ] 统一 AI 客户端（ai/client.ts）— 支持9家提供商
- [ ] 文章分析模块（ai/analyzer.ts）— Prompt + JSON解析
- [ ] 微调面板（StylePanel）— 5根滑条
- [ ] 滑条 → CSS参数映射逻辑
- [ ] "恢复AI推荐"按钮
- [ ] localStorage 存储 API 配置
- [ ] 无 API Key 时的优雅降级

### 验收标准

1. 配置千问/DeepSeek API Key后，粘贴文章点击"AI分析"，3-10秒内返回推荐方案
2. 推荐方案的配色/排版/装饰/字体与文章气质匹配
3. 拖动滑条，预览实时更新（<200ms延迟）
4. 不配置API Key，所有核心功能正常可用

### 交付清单

- [ ] 尾图模板

- [ ] 小红书 HTML 渲染器（render/xiaohongshu.ts）
- [ ] 小红书预览组件（XhsPreview）
- [ ] html2canvas 集成 — HTML→PNG
- [ ] 单张下载 + ZIP打包下载
- [ ] 图片尺寸选择（3:4 / 1:1 / 16:9）

### 验收标准

2. 封面图有大标题，尾图有引导关注
3. 自定义字体（如得意黑、霞鹜文楷）在图片中正确显示
4. 导出的PNG尺寸为1080×1440（3:4），文字清晰

# V2 版本：骨架 × 插槽 开发分期追加

> V2 在 V1 Phase 1-4 基础上新增 Phase 5-7，实现骨架+插槽排版引擎。
> V1 的所有 Phase 保持不变，V2 Phase 从 Phase 5 开始编号。

---

## V2 总体规划

| 阶段 | 周期 | 核心交付 | 对应功能 |
|------|------|---------|---------|
| **Phase 5: 骨架引擎** | 1-2周 | 骨架注册表 + 渲染调度 + 基础UI | F11 |
| **Phase 6: 插槽系统** | 2周 | 10维插槽 + 协调引擎 + 完整面板 | F12+F13+F14+F15 |
| **Phase 7: AI协调增强** | 1周 | AI插槽推荐 + 预设导入导出 | F15增强 |

---

## Phase 5: 骨架引擎（第一批）

**目标**: 15种骨架可切换，渲染引擎从平铺循环改为骨架调度

### 交付清单

- [ ] `blueprints.ts` — 15种骨架注册表（每种含 wrapDocument + wrapSection 渲染函数）
- [ ] `slots/title.ts` — 标题插槽 20+变体
- [ ] `slots/quote.ts` — 引用插槽 15+变体
- [ ] `slots/list.ts` — 列表插槽 12+变体
- [ ] `slots/divider.ts` — 分割线插槽 15+变体
- [ ] `slots/paragraph.ts` — 段落插槽 8+变体
- [ ] `slots/section.ts` — 节区插槽 10+变体
- [ ] `slots/index.ts` — 插槽注册表总入口 + 查询/解析函数
- [ ] `atoms/index.ts` 改写 — 扩展 StyleCombo 为 StyleComboV2
- [ ] `render/wechat.ts` 改写 — 骨架调度器（groupBySection + renderNode）
- [ ] `LayoutPanel.tsx` — 骨架网格选择面板 + 6个核心插槽下拉
- [ ] `App.tsx` 更新 — 集成骨架切换 + 插槽面板
- [ ] 向后兼容：V1 StyleCombo 自动升级为 V2
- [ ] 微信公众号粘贴测试：15种骨架全部通过

### 验收标准

1. 15种骨架都能正确渲染，切换<100ms
2. 6个核心插槽各变体都能独立切换
3. 骨架切换自动加载默认插槽搭配
4. 复制富文本到微信公众号编辑器，所有骨架样式正确保留
5. V1 的原有功能不受影响

---

## Phase 6: 插槽系统 + 协调引擎（第二批）

**目标**: 4个装饰性插槽 + 锁定机制 + 协调引擎 + 完整 UI

### 交付清单

- [ ] `slots/frame.ts` — 边框装饰 9+变体
- [ ] `slots/ornament.ts` — 装饰花纹 6+变体
- [ ] `slots/callout.ts` — 提示框 6+变体
- [ ] `slots/badge.ts` — 标签徽章 6+变体
- [ ] `coordinator.ts` — 本地规则协调引擎（tags匹配 + affinity得分）
- [ ] 锁定机制 — 每个插槽 🔒/🔓 切换
- [ ] 调节模式切换 — 自由调节 / AI协调
- [ ] `SlotSelector.tsx` — 通用插槽选择器组件（下拉 + 预览 + 锁定按钮）
- [ ] `LayoutPanel.tsx` 扩展 — 10个插槽全部展示
- [ ] 每个变体添加 tags 和 affinity 数据
- [ ] 锁定状态 localStorage 持久化
- [ ] 协调引擎性能优化（<50ms）

### 验收标准

1. 10个维度插槽全部可独立切换
2. 锁定某插槽后切换骨架，该插槽保持不变
3. 协调模式下修改一个插槽，其他未锁定插槽自动匹配
4. 协调结果视觉和谐（人工抽样测试）
5. 自由模式下各插槽互不影响

---

## Phase 7: AI 协调增强（第三批）

**目标**: AI 智能推荐 + 品牌预设 + 用户体验优化

### 交付清单

- [ ] AI 协调模式 — 调用 LLM 分析当前搭配给出推荐
- [ ] AI 分析增强 — V2 Prompt 返回骨架+插槽推荐（不只是原子）
- [ ] 品牌预设保存 — 用户保存常用的骨架+插槽组合
- [ ] 品牌预设导入/导出（JSON格式）
- [ ] "随机搭配"增强 — 基于协调引擎生成和谐随机组合
- [ ] 骨架缩略图生成 — 每种骨架的小预览图
- [ ] 插槽变体预览 — 鼠标悬浮显示效果预览
- [ ] 移动端适配 — 插槽面板折叠/展开优化
- [ ] 无 API Key 降级 — AI协调失败自动回退本地规则

### 验收标准

1. AI协调能给出合理的插槽推荐
2. AI失败时无感回退到本地协调
3. 品牌预设可保存/加载/导出/导入
4. 移动端面板交互流畅
