# 我做了一个开源排版工具，让公众号和小红书排版从30分钟变成30秒

> 作为一个经常写公众号的人，我受够了排版这件事。于是花了3天，用 React + TypeScript 做了一个完全免费的 AI 排版引擎——云中书 YunType。

---

## 排版，创作者的噩梦

如果你写过公众号，一定经历过这样的场景：

文章写了2个小时，排版又花了1个小时。字号大小、行间距、配色、标题样式……每一个细节都要调。更崩溃的是，你精心调好的排版，换一台电脑打开就全乱了。

小红书更离谱。你需要把文字手动切成多张图片，每张都要保持一致的风格。一篇3000字的文章，可能要做10-15张图。光是对齐文字就够喝一壶的。

市面上的排版工具呢？

- **秀米/135编辑器**：模板多但同质化严重，高级功能要付费
- **Canva**：偏设计工具，对纯文字排版支持弱
- **手动CSS**：技术门槛高，普通创作者用不了

我想要的很简单：**粘贴文章，选个风格，一键复制到微信公众号。如果是小红书，直接帮我切成图片组下载。**

于是我自己做了一个。

---

## 云中书：660种排版，一个也不重复

**云中书 YunType** 的核心理念是**原子化组合**。

我没有做"模板"。模板的问题是：好看的就那几个，用的人多了就撞款。

我做的是**四个维度的自由组合**：

```
排版方案 = 配色(11种) × 排版(5种) × 装饰(4种) × 字体(3种)
         = 660 种独特组合
```

### 四个维度分别是什么？

**🌈 配色（11种）**

8套浅色系 + 3套深色系。从奶茶温柔到深空科技，从樱花浪漫到暗夜酒红。每套配色不只是一个主色，而是一整套包含标题色、正文色、背景色、引用色、代码色的完整色彩系统。

**📐 排版（5种）**

- 紧凑知识型：适合技术教程，信息密度高
- 舒适阅读型：适合深度长文，大量留白
- 杂志画报型：首字下沉，视觉冲击
- 清单列表型：适合攻略评测，信息模块化
- 对话访谈型：对话气泡风格

**🎭 装饰（4种）**

极简线条、卡片容器、渐变装饰、几何装饰。决定文章的"呼吸感"和"设计感"。

**✒️ 字体（3种基础 + 10种可选）**

现代简约、典雅宋体、活泼趣味。另外还有10种可选字体，按需从CDN加载，标题和正文可以分别选择。

这意味着：**你选的每一种组合，在创作者圈子里几乎不会和别人撞款**。

## 不只是公众号：小红书模式才是杀手锏

公众号排版是基础功能。真正让我自己用得停不下来的是**小红书模式**。

### 它怎么工作的？

1. **粘贴你的 Markdown 文章**
2. **自动分页**：智能按段落切分成 1080×1440 的图片页
3. **5种封面变体**：经典、大字、极简、卡片、杂志风格随机生成
4. **拖拽排序**：生成后可以拖拽调整页面顺序
5. **一键下载**：所有图片打包成 ZIP，自动编号

每一页都继承你选择的配色、排版、装饰和字体。**整套图片的视觉一致性非常高**，比手动做 Canva 模板快10倍不止。

尾页还自带互动引导（点赞/收藏/转发按钮）和品牌区域。

## AI 加持：让机器帮你选风格

选择困难症？没关系。

云中书集成了 **9 个 AI 提供商**的文章分析能力：

- 通义千问、DeepSeek、豆包、OpenAI、Gemini、Moonshot、智谱、SiliconFlow
- 还支持自定义 OpenAI 兼容接口

**工作原理**：AI 会分析你文章的主题、文风、目标受众，然后从 660 种组合中推荐最匹配的排版方案。

比如：
- 一篇科技评测 → 推荐「烟灰高级 + 紧凑知识 + 卡片容器 + 现代简约」
- 一篇旅行游记 → 推荐「天青雅致 + 杂志画报 + 渐变装饰 + 典雅宋体」
- 一篇美食分享 → 推荐「奶茶温柔 + 舒适阅读 + 极简线条 + 活泼趣味」

**重要的是**：你的 API Key 直接调用提供商 API，数据不经过任何中间服务器。没有 Key 也能用——离线时自动切换到本地规则分析。

## 更多你可能想不到的功能

### 🌙 暗黑模式

Ctrl+D 一键切换，深色界面保护眼睛。设置自动保存，刷新不丢失。

### ⌨️ 键盘快捷键

- `Ctrl + Shift + R`：随机生成一套排版组合（手感超爽）
- `Ctrl + E`：导出当前排版
- `Ctrl + D`：切换暗黑模式

### 💼 品牌预设

找到满意的排版后，保存为品牌预设（最多20套）。下次写文章直接调用，保持品牌一致性。

### 🕐 历史记录

自动保存最近10次排版配置，随时回溯。

### 📊 信息图模式

内置4种信息图模板：流程图、对比表、知识卡片、时间线。用 Markdown 写数据，程序化渲染成精美信息图。

### 🎨 AI 文生图

集成通义万相、豆包、OpenAI DALL-E、Gemini 四个图片生成提供商，一键生成文章配图。

### 📱 移动端适配

平板、手机、超小屏全适配。在手机上也能排版（虽然我建议用电脑效果更好）。

### PWA 支持

可以添加到手机/电脑桌面，像 App 一样使用。Service Worker 实现离线缓存。

## 技术细节（给程序员看的）

如果你是开发者，可能会对技术实现感兴趣：

| 技术 | 选型 | 理由 |
|------|------|------|
| React 19 + TypeScript | 组件化 + 类型安全 | |
| Vite 6 | 极速 HMR，开发体验好 | |
| 纯内联 CSS | 100% 微信公众号兼容 | 微信不支持外部 CSS |
| 自研 Markdown 解析器 | 精准控制 AST，不依赖 marked/remark | |
| html2canvas | 小红书模式 HTML→Canvas→PNG | 动态加载，不影响首屏 |
| JSZip | 图片批量打包 | 动态加载 |
| Service Worker | PWA 离线支持 | 网络优先策略 |

### 构建产物

```
dist/index.js      310KB (gzip: 94KB)   主包
dist/index.css       2.5KB (gzip: 0.8KB)  样式
html2canvas.esm.js  202KB (gzip: 48KB)   懒加载
jszip.min.js         98KB (gzip: 30KB)   懒加载
```

首屏只加载 ~95KB（gzip），html2canvas 和 JSZip 通过 `dynamic import()` 在需要时才加载。

整个项目**零 UI 组件库依赖**（没有 Ant Design / Material UI），所有 UI 都是纯内联 CSS 手写的。
