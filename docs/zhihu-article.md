# 云中书 YunType：一个开源AI排版引擎的诞生——660种排版组合，让公众号和小红书创作效率翻10倍

> 我是一个经常写公众号和小红书的创作者。每次文章写完，排版这件事就让我抓狂。于是我用 React + TypeScript 从零做了一个完全免费、完全开源的 AI 排版引擎——**云中书 YunType**。这篇文章会详细介绍它的每一个功能、设计理念和技术实现。

## 一、为什么要做这个工具？

### 1.1 公众号排版的痛

如果你写过微信公众号文章，一定经历过这样的场景：

花了 2 个小时写完一篇 3000 字的深度文章，打开公众号后台，开始排版。字号要调、行间距要改、标题样式要统一、配色要协调……又花了 1 个小时。更崩溃的是，你精心调好的排版，换一台手机打开发现字体全变了。

你可能试过市面上的排版工具：

| 工具 | 优点 | 痛点 |
|------|------|------|
| **秀米** | 模板多 | 同质化严重，高级功能要付费，导出有水印 |
| **135编辑器** | 组件丰富 | 学习曲线陡峭，会员制收费 |
| **Canva** | 设计感强 | 偏图片设计，纯文字排版弱 |
| **手写CSS** | 完全自由 | 技术门槛高，普通创作者用不了 |
| **Notion导出** | Markdown友好 | 样式固定，无法自定义 |

### 1.2 小红书排版的痛

如果说公众号排版是"折磨"，小红书排版就是"酷刑"。

小红书是图片平台，你的文字文章需要变成一组精美的图片。一篇 3000 字的攻略，你可能需要：

- 手动把文字分成 10-15 张图
- 每张图保持一致的背景色、字体、排版风格
- 封面要有冲击力
- 尾页要有互动引导

用 Canva 做一套大概需要 30-60 分钟，而且每次换个风格就要重来。

### 1.3 我想要的很简单

> **粘贴 Markdown 文章 → 选个风格 → 一键复制到公众号。如果是小红书，直接帮我把文章自动切成一组图片，打包下载。**

于是我自己做了一个。

## 三、核心功能：660种原子化排版组合

### 3.1 什么是"原子化组合"？

传统排版工具提供"模板"。模板的问题是：好看的就那几个，用的人多了就撞款。你的文章和别人看起来一模一样。

云中书不做模板。我做的是**四个维度的自由组合**：

```
排版方案 = 配色方案(11种) × 排版结构(5种) × 装饰风格(4种) × 字体气质(3种)
         = 660 种独特组合
```

每个维度独立选择，互不影响。就像搭配衣服一样——上衣、裤子、鞋子、配饰各选一件，你的穿搭就是独一无二的。

| `pageBg` | 页面背景色 |

### 4.2 智能分页算法

这是小红书模式的技术核心。系统会：

1. 解析 Markdown 为 AST（抽象语法树）
2. 估算每个元素（标题/段落/列表/代码块）在画布上的高度
3. 按可用高度自动切分成多页
4. 第一页永远是**封面**，最后一页是**尾页**

高度估算公式：
```
估算高度 = Math.ceil(文字长度 / 每行字数) × 字号 × 行高
```

比如一段 300 字的正文，在 1080×1440 画布上（padding 60px，字号 32px，行高 1.8）：
```
可用宽度 = 1080 - 60×2 = 960px
每行字数 = 960 / 32 = 30字
行数 = Math.ceil(300/30) = 10行
估算高度 = 10 × 32 × 1.8 = 576px
```

### 4.3 五种封面变体

每次生成，封面会从 5 种变体中随机选择：

| 变体 | 风格描述 |
|------|----------|
| 经典 | 大标题居中，副标题在下方 |
| 大字 | 超大字号标题，视觉冲击 |
| 极简 | 简洁留白，标题+一条线 |
| 卡片 | 半透明卡片容器包裹标题 |
| 杂志 | 上图下文的杂志封面风格 |

### 4.4 拖拽排序

生成后的页面顺序不满意？直接在缩略图上**拖拽排序**！拖动缩略图到目标位置，松手即完成。

## 五、AI 加持：让机器帮你选风格

### 5.1 AI 文章分析

选择困难症？让 AI 帮你选。

点击顶栏的 **🤖 AI分析** 按钮，会弹出 AI 配置对话框。输入你的 API Key，选择提供商，AI 会分析你文章的：

- **文章类型**（技术教程/情感随笔/商业分析/旅行游记…）
- **文章气质**（严肃/温暖/活泼/专业…）
- **目标受众**

然后从 660 种组合中推荐最匹配的排版方案，并给出推荐理由。

比如：
- 你写了一篇**编程教程** → AI 推荐：`极夜极光 + 紧凑知识型 + 几何装饰 + 现代简约`（理由：技术类文章适合深色系，紧凑排版信息密度高）
- 你写了一篇**旅行游记** → AI 推荐：`樱花物语 + 杂志编辑型 + 圆润气泡 + 文艺优雅`（理由：旅行文章适合浪漫配色和杂志质感）

### 8.1 技术选型

| 技术 | 版本 | 选择理由 |
|------|------|----------|

| **html2canvas** | — | 小红书模式 HTML→Canvas→PNG |
| **JSZip** | — | 图片批量打包下载 |
| **Service Worker** | — | PWA 离线缓存 |

### 8.2 为什么用纯内联 CSS？

这是一个**非常关键的技术决策**。

微信公众号的编辑器对 CSS 有严格限制：
- ❌ 不支持 `<style>` 标签
- ❌ 不支持外部 CSS 文件
- ❌ 不支持 CSS class
- ✅ 只支持 `style="..."` 内联样式

所以云中书生成的所有 HTML **全部使用内联 style 属性**。没有一行 CSS class。这保证了复制到公众号后排版 100% 还原。

### 8.3 为什么自研 Markdown 解析器？

市面上有很多优秀的 Markdown 解析库（marked、remark、markdown-it），但我选择自研，原因是：

1. **精准控制 AST 结构**：我需要在解析阶段就为每个节点注入完整的内联样式
2. **体积更小**：自研解析器只实现需要的语法子集，不到 200 行代码
3. **性能更好**：不需要先解析再后处理，一次遍历完成
4. **适配小红书分页**：解析器输出的 AST 可以直接用于高度估算和分页算法

支持的语法：标题(h1-h3)、段落、粗体、斜体、删除线、链接、图片、有序/无序列表、引用块、代码块、行内代码、表格、分割线。

### 8.4 懒加载策略

首屏只加载核心代码，重量级的第三方库通过 `dynamic import()` 按需加载：

```
首屏加载（必须）:
├── React + ReactDOM    ~45KB (gzip)
├── App + 组件 + 原子引擎   ~50KB (gzip)
└── 总计 ~95KB (gzip)

按需加载（用到才加载）:
├── html2canvas    202KB → 48KB (gzip)   ← 点击"导出小红书图片"时
├── JSZip           98KB → 30KB (gzip)   ← 点击"全部导出ZIP"时
└── 各AI提供商SDK   按需                   ← 配置API Key时
```

### 8.6 零 UI 组件库

## 十、实际使用场景

### 场景一：自媒体日常更新

**小王**是一个科技博主，每周写3篇公众号文章。

- **以前**：写完文章后，在135编辑器里花30分钟排版，每次都要重新调样式
- **现在**：打开云中书，粘贴 Markdown，选择"商务简约"预设，3秒搞定。用品牌预设功能保存了自己的风格，下次直接调用

**效率提升**：每篇文章节省 30 分钟 → 每周节省 1.5 小时

### 场景二：小红书种草博主

**小李**是一个美食博主，需要把探店文章发到小红书。

- **以前**：在 Canva 里手动做10张图，每张调字体、调颜色、调排版，花1小时
- **现在**：切到小红书模式，选"奶茶温柔"配色 + 3:4比例，自动生成10页图片。拖拽调整顺序后一键下载ZIP

**效率提升**：从1小时变成3分钟

### 场景三：运营团队品牌统一

**某品牌**有5个公众号运营，每个人排版风格都不一样。

- **以前**：写排版规范文档，但每个人理解不同，最终效果参差不齐
- **现在**：品牌负责人在云中书调好排版 → 保存为品牌预设 → 分享预设代码 → 所有人一键应用

## 十一、开源故事

云中书（YunType）这个名字取自李清照《一剪梅》：

> *云中谁寄锦书来？雁字回时，月满西楼。*

"锦书"是古代用锦缎写成的精美书信。我希望每一篇用云中书排版的文章，都像一封精美的锦书。

项目 **MIT 协议开源**，意味着：
- ✅ 个人和商业使用都完全免费
- ✅ 可以修改、分发、私有部署
- ✅ 没有付费墙、没有 Pro 版本
- ✅ 没有"高级功能需要登录"
