# 布局系统架构计划

## 一、核心问题

用户看到了15种差异化布局后，接下来要解决三个问题：

1. **怎么组合** — 15种骨架 + 每个元素的变体，如何让用户选择和切换？
2. **怎么调整** — 用户想改某个元素（比如只改标题样式），怎么操作？
3. **AI怎么配合** — 改了一个元素后，AI如何自动协调其他元素？又如何尊重用户"这个不要动"的意愿？

---

## 二、Slot 插槽系统设计

### 2.1 将每种布局拆成 6 个独立插槽

```
一个完整版式 = 骨架(Blueprint) + 6个元素插槽(Slot)

骨架决定：内容分区逻辑（线性/卡片/时间线/交替色带/...）
插槽决定：每个元素的具体视觉表现
```

| 插槽 | 说明 | 变体数量 | 示例 |
|------|------|---------|------|
| `title` | 标题样式 | 10+ | 左对齐底线 / 居中 / 色块标签 / 双线框 / Banner / 编号序号 / 花纹装饰 / 左色条 / 几何前缀 / 日式细字 |
| `quote` | 引用块样式 | 8+ | 左竖线 / 圆角卡片 / Pull-quote居中 / 双线框 / 气泡 / 大引号 / 虚线框 / 摘要框 |
| `list` | 列表样式 | 8+ | ● 圆点 / ■ 方块 / ▶ 箭头 / 🔹 菱形 / ①②③ / ☑ 清单 / → 箭头 / — 短线 |
| `divider` | 分割线样式 | 7+ | 细线 / 粗线 / 双线 / 渐变线 / 点状 · · · / 装饰花 ❖ / 菱形 ◇ |
| `paragraph` | 段落样式 | 5+ | 无缩进紧凑 / 首行缩进 / 首字下沉 / 大留白 / 首段放大 |
| `section` | 节区包裹 | 5+ | 平铺无包裹 / 卡片阴影 / 交替色带 / 时间线轨道 / 无分割自然流 |

### 2.2 骨架 vs 插槽的关系

骨架是"推荐搭配"——每种骨架预设了一套默认插槽组合（已经调好看了）。

```typescript
// L05 日式留白骨架的默认配置
const L05: Blueprint = {
  id: 'L05',
  name: '日式留白',
  defaultSlots: {
    title: 'zen-minimal',      // 居中细字体
    quote: 'dashed-frame',     // 虚线框
    list: 'dash-prefix',       // — 短线前缀
    divider: 'single-circle',  // 居中 ○
    paragraph: 'airy-wide',    // 超大留白行高
    section: 'flat-flow',      // 平铺无包裹
  }
}
```

用户选骨架 → 默认插槽全部就位 → 开箱好看。
然后用户可以**单独换任何一个插槽**。

## 三、用户操作界面设计

### 3.1 两级交互

```
第一级：选骨架（大类切换）
  ┌──────────────────────────────────────┐
  │  [极简] [学术] [散文] [日式] [杂志]    │
  │  [色块] [双线] [卡片] [色带] [时间线]   │
  │  [对话] [报刊] [科技] [首字] [左色条]   │
  └──────────────────────────────────────┘
  点击一个 → 整篇文章结构一次性变化

第二级：调插槽（局部微调）
  展开当前骨架的插槽面板：
  ┌─────────────────────────────────────────┐
  │ 🔒 标题   [居中细字 ▾]    ← 可换下拉    │
  │ 🔓 引用   [虚线框   ▾]    ← 可换下拉    │
  │ 🔓 列表   [短线前缀 ▾]    ← 可换下拉    │
  │ 🔓 分割线 [居中圆   ▾]    ← 可换下拉    │
  │ 🔓 段落   [大留白   ▾]    ← 可换下拉    │
  │ 🔓 节区   [平铺     ▾]    ← 可换下拉    │
  │                                         │
  │ [🔒 锁定全部] [🔓 解锁全部]              │
  │ [🎲 随机全部] [🤖 AI协调]               │
  └─────────────────────────────────────────┘
```

### 3.2 🔒 锁定机制

每个插槽旁边有一个小锁图标：
- **🔓 解锁**（默认）：可以被"随机"或"AI协调"改变
- **🔒 锁定**：该插槽的值被保护，任何自动操作都不会改它

**使用场景**：
- 用户很喜欢当前的标题样式 → 锁定标题
- 然后点"AI协调" → AI 只调整未锁定的其他5个插槽
- 或者点"随机" → 只随机未锁定的插槽

## 四、两种调整模式

### 4.1 🎨 自由调节模式（默认）

**行为**：改一个插槽，其他插槽不动。

```
用户把引用从"虚线框"改成"大引号" 
  → 只有引用块变了
  → 标题、列表、分割线、段落、节区全部不变
```

**适合**：知道自己想要什么的用户，精确控制。

### 4.2 🤖 AI 协调模式

**行为**：改一个插槽后，AI 自动调整其他**未锁定**的插槽，使整体风格协调。

```
用户把引用从"虚线框"改成"大引号"
  → AI 分析："大引号"偏文艺/装饰感
  → 自动将标题调为"花纹装饰" (如果未锁定)
  → 自动将分割线调为"装饰花❖" (如果未锁定)  
  → 自动将列表调为"圆点●" (如果未锁定)
  → 锁定的插槽不动
```

### 4.3 AI 协调的实现方案

#### 方案A：规则引擎（无需API，本地运行）

为每个插槽变体打上风格标签：

```typescript
const slotMeta = {
  'big-quotes': { 
    tags: ['literary', 'decorative', 'elegant'],
    affinity: { // 亲和力映射
      title: ['floral', 'center', 'zen-minimal'],
      divider: ['ornament', 'dots'],
      list: ['dot', 'dash-prefix'],
      paragraph: ['indented', 'drop-cap'],
    }
  },
  'dashed-frame': {
    tags: ['minimal', 'japanese', 'clean'],
    affinity: {
      title: ['zen-minimal', 'left-simple'],
      divider: ['single-circle', 'thin-line'],
      list: ['dash-prefix', 'dot'],
      paragraph: ['airy-wide', 'indented'],
    }
  },
  // ...
}
```

当用户改了一个插槽：
1. 读取新选项的 `tags`
2. 对每个未锁定的其他插槽，找 `affinity` 中匹配的变体
3. 如果找不到，按 tags 相似度排序选最佳匹配
4. 立即应用，实时预览

**优点**：零延迟、无需网络、可预测
**缺点**：需要预先标注所有亲和关系

#### 方案B：AI API 调用

把当前所有插槽配置 + 用户改动 + 锁定状态发给 AI：

```
Prompt: 
当前版式配置：标题=居中细字, 引用=虚线框, 列表=短线, 分割=圆, 段落=大留白, 节区=平铺
用户将引用改为：大引号
锁定的插槽：标题（不可改）
请推荐其他未锁定插槽的最佳搭配，从以下选项中选择...
```

**优点**：灵活、能理解上下文
**缺点**：有延迟、需要 API Key

#### 推荐：A + B 混合

- **默认用规则引擎**（方案A）：改插槽后 < 100ms 出结果
- **可选 AI 增强**（方案B）：点"🤖 AI深度协调"按钮，调 API 获得更精准建议
- AI 结果可以覆盖规则引擎的结果，用户确认后生效

## 五、UI 交互流程

### 5.1 完整用户旅程

```
1. 用户粘贴文章
         ↓
2. 选择骨架（15选1）→ 默认插槽全部就位 → 预览立刻好看
         ↓
3. [可选] 切换调整模式：
   ┌─────────────────────────┐
   │ 🎨 自由调节  │ 🤖 AI协调 │  ← 顶部切换开关
   └─────────────────────────┘
         ↓
4. [可选] 锁定某些插槽
         ↓
5. [可选] 调整单个插槽 → 
   - 自由模式：只改这一个
   - AI模式：自动协调其他未锁定的
         ↓
6. [可选] 微调 CSS 参数（字号倍率、间距倍率、行高）
         ↓
7. 满意 → 导出
```

### 5.2 快速操作

| 操作 | 效果 |
|------|------|
| 🎲 随机骨架 | 换一个骨架 + 全部默认插槽 |
| 🎲 随机插槽 | 保持当前骨架，随机所有未锁定插槽 |
| 🤖 AI 推荐 | 根据文章内容，推荐骨架 + 插槽 + 配色 |
| ↩️ 恢复默认 | 把所有插槽恢复为当前骨架的默认值 |

## 六、数据结构

```typescript
// 完整版式配置
interface LayoutConfig {
  blueprintId: string     // 骨架ID: 'L01' ~ 'L15'
  slots: {
    title: string         // 标题变体ID
    quote: string         // 引用变体ID  
    list: string          // 列表变体ID
    divider: string       // 分割线变体ID
    paragraph: string     // 段落变体ID
    section: string       // 节区变体ID
  }
  locks: {
    title: boolean        // 是否锁定
    quote: boolean
    list: boolean
    divider: boolean
    paragraph: boolean
    section: boolean
  }
  mode: 'free' | 'ai-coordinated'  // 调整模式
}

// 插槽变体注册表
interface SlotVariant {
  id: string
  name: string
  tags: string[]          // 风格标签
  affinity: Record<string, string[]>  // 对其他插槽的亲和推荐
  render: (content: string, colors: Colors, ...args) => string  // 渲染函数
  preview: string         // 缩略图或简短预览HTML
}

// 骨架
interface Blueprint {
  id: string
  name: string
  desc: string
  icon: string
  defaultSlots: Record<string, string>  // 默认插槽配置
  render: {
    // 骨架的结构渲染函数
    groupSections: (nodes: MarkdownNode[]) => Section[]
    wrapDocument: (sectionsHtml: string, colors: Colors) => string
    wrapSection: (sectionHtml: string, heading: string, colors: Colors, idx: number) => string
  }
}
```

## 八、关键设计决策

### Q1：骨架和插槽能否完全自由组合？
**答**：大部分可以，但有些不兼容。例如"时间线"骨架的节区必须是"timeline"，不能换成"卡片"。通过 `Blueprint.compatibleSlots` 限制不兼容项。

### Q2：插槽变体的渲染函数怎么与骨架配合？
**答**：骨架只负责"在哪里"放内容（结构），插槽负责"怎么渲染"内容（样式）。骨架调用插槽的 render 函数，传入内容和配色。

### Q3：用户的自定义配置怎么保存？
**答**：扩展现有的品牌预设系统。一个品牌预设 = 骨架ID + 6个插槽ID + 配色ID + 字体ID + 微调参数。JSON 格式，存 localStorage，可导出分享。

### Q4：AI协调是实时的还是需要确认？
**答**：规则引擎实时生效（改了立刻变），AI API 调用需要确认（弹出推荐面板，用户选择接受/拒绝）。
