# 云中书 / YunType — 产品概述

> **版本**: v1.0 | **最后更新**: 2026-04-10 | **状态**: 规划中

---

## 1. 产品定义

**云中书（YunType）** 是一款 AI 驱动的内容排版工具，帮助公众号和小红书创作者**不需要设计能力，也能做出专业、好看、有品牌辨识度的排版**。

### 一句话定位

> "粘贴文章 → AI 自动推荐排版 → 一键导出"

### 核心价值主张

| 用户痛点 | 云中书的解法 |
|---------|-----------|
| 不会设计，不知道选什么模板 | AI 分析文章内容和文风，自动推荐最匹配的排版方案 |
| 135/壹伴模板用烂了，所有人长得一样 | 原子化设计系统，660+ 种组合可能，不会撞款 |
| 想要品牌感，但不知道怎么做 | 滑条式微调面板，调"感觉"而不是调参数 |
| 只服务公众号，小红书还得另找工具 | 同时支持公众号富文本和小红书图片组 |

---

## 2. 目标用户

### 主要用户画像

**画像A：个人公众号创作者**
- 每周发1-3篇文章
- 会写内容，但不会设计
- 用过135/壹伴的免费模板，觉得不够好看或太同质化
- 对排版的需求："好看就行，别让我选太多"
- 技术水平：会复制粘贴，不懂 HTML/CSS

**画像B：小红书知识类博主**
- 发"干货帖"——把知识点做成图片组
- 目前用 Canva/稿定设计做图，觉得太慢
- 希望"输入文字 → 自动生成一组图片"
- 对排版的需求："快、好看、有统一的风格"

**画像C：企业自媒体运营**
- 同时管理公众号+小红书
- 需要品牌一致性（统一配色、统一风格）
- 发文频率高，效率是第一需求
- 对排版的需求："统一品牌风格，快速套用"

### 非目标用户

- 专业设计师（他们不需要这个工具）
- 追求极致自定义的高级用户（应该用 Figma/Sketch）
- 纯文字无排版需求的用户

## 3. 竞品分析

### 直接竞品

| 竞品 | 优势 | 劣势 | 云中书的差异化 |
|------|------|------|-------------|
| **135编辑器** | 海量模板（10000+），微信完美兼容 | 模板同质化严重，选择恐惧，免费版限制多 | 原子组合660+种，AI代选，免费开源 |
| **壹伴** | Chrome插件直接在微信编辑器内使用，方便 | 模板有限，风格趋同 | 独立工具，不依赖微信编辑器 |
| **秀米** | 可视化拖拽编辑，交互好 | 学习成本高，深度使用需付费 | 零学习成本，粘贴即用 |

### 间接竞品

| 竞品 | 类别 | 云中书的差异 |
|------|------|-----------|
| **Canva** | 通用设计工具 | 云中书专注排版，更快更简单 |
| **稿定设计** | 模板设计工具 | 云中书有AI推荐，不需要手动选模板 |
| **ChatGPT/Claude 直接生成HTML** | AI生成 | 每次风格不一致，无品牌体系，不保证微信兼容 |

### 云中书的独特壁垒

1. **原子化设计系统** — 不是模板库，是可参数化组合的设计系统
2. **AI 推荐 + 滑条微调** — 用户操作"感觉"而非"选择"
3. **免费开源** — 135/壹伴不会做免费开源（会蚕食付费用户）
4. **公众号+小红书双模式** — 一个工具覆盖两个平台

## 4. 产品形态

### 技术形态

- **纯前端应用**（无后端服务器）
- **单页应用（SPA）**，使用 React + TypeScript
- **静态部署**于 Vercel / GitHub Pages / Netlify
- **用户自带 API Key**（用于 AI 分析和生图功能）
- **离线可用**的核心功能（排版预览、微调、导出不需要网络）

### 技术栈选型

| 层 | 技术 | 理由 |
|---|------|------|
| 框架 | React + TypeScript | 与 InfiniteSkill 保持一致，组件生态成熟 |
| 构建 | Vite | 快速，已有经验 |
| 样式 | Tailwind CSS（工具界面） + 纯内联CSS（排版输出） | 工具界面用现代CSS，输出必须微信兼容 |
| 图片生成 | html2canvas（首选），modern-screenshot（备选） | 纯前端 HTML→PNG 转换，零后端依赖 |
| 字体加载 | Google Fonts CDN + jsDelivr CDN | 小红书模式需要自定义字体 |
| 部署 | Vercel | 已有部署经验 |

### 不需要 API Key 的功能（80%）

- 粘贴文章
- 手动选择配色/排版/装饰/字体组合
- 预览排版效果
- 微调滑条
- "换一个"随机组合
- 复制富文本 / 导出HTML（公众号模式）
- 生成并下载图片组（小红书模式）

### 需要 API Key 的功能（20%）

- AI 分析文章 → 自动推荐排版方案
- AI 文生图（配图生成）
- AI 拆分文章为小红书图片组

## 5. 核心用户流程

### 公众号模式

```
用户粘贴文章
    ↓
选择平台：[公众号] / 小红书
    ↓
[有API Key] AI分析 → 推荐方案    [无API Key] 随机推荐一个方案
    ↓                                ↓
预览排版效果 ←────────────────────────┘
    ↓
满意？→ 是 → 复制富文本 → 粘贴到微信编辑器
  ↓ 否
微调面板（滑条） / "换一个"按钮
    ↓
重新预览 → 满意 → 复制/导出
```

### 小红书模式

```
用户粘贴文章/输入要点
    ↓
选择平台：公众号 / [小红书]
    ↓
[有API Key] AI拆分 → 分配到多张图    [无API Key] 用户手动分段
    ↓                                    ↓
选择图片尺寸：[3:4] / 1:1 / 16:9
    ↓
预览图片组（封面图 + 内容图 + 尾图）
    ↓
微调风格 / 换配色 / 换字体
    ↓
下载图片组（PNG）→ 上传到小红书
```

## 7. 设计原则

基于 Frontend Design Skill 的核心理念，云中书遵循以下设计原则：

### 工具界面设计原则

1. **反 AI 审美** — 禁止 Inter/Roboto 字体、禁止紫色渐变白底、禁止千篇一律的卡片布局
2. **大胆的审美方向** — 工具本身要有辨识度，要让人记住
3. **Typography 驱动** — 用独特的字体搭配建立品牌感
4. **有意图的设计** — 每个设计决策都要有明确的原因

### 排版输出设计原则

1. **协调性约束** — 用户不管怎么调滑条，出来的结果都是协调的
2. **多样性** — 660种组合，每种都有独特的性格
3. **微信兼容优先** — 公众号模式的所有CSS必须在微信编辑器中可用
4. **字体即设计** — 小红书模式中，字体是核心差异化元素
