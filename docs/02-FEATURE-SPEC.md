## 2. 各功能详细规格

### F1: 文章输入

**优先级**: P0（必须）

**描述**: 用户将文章内容粘贴到输入区域。

**交互规格**:
- 一个大的文本输入区（textarea），支持纯文本粘贴
- 支持 Markdown 格式识别（标题 `#`、列表 `-`、引用 `>`、粗体 `**`、代码块等）
- 输入区提示文字："粘贴你的文章内容，支持 Markdown 格式…"
- 右上角显示字数统计
- 支持快捷键 Ctrl+V 粘贴

**技术实现**:
- `<textarea>` 或富文本编辑器（推荐先用 textarea + Markdown 解析）
- Markdown 解析库：`marked` 或 `markdown-it`
- 字数统计：实时计算，中文按字计数

**验收标准**:
- [ ] 可以粘贴纯文本
- [ ] 可以识别 Markdown 标题（h1-h4）
- [ ] 可以识别列表（有序/无序）
- [ ] 可以识别引用、粗体、代码块
- [ ] 字数统计实时更新

### F2: 平台选择

**优先级**: P0（必须）

**描述**: 用户选择目标发布平台，决定输出格式。

**交互规格**:
- 两个选项卡/按钮：`📝 公众号` 和 `📸 小红书`
- 默认选中"公众号"
- 切换平台时，下方的预览和设置面板自动更新
- 小红书模式下额外显示：图片尺寸选择（3:4 / 1:1 / 16:9）

**技术实现**:
- 简单的 tab 切换状态
- 全局状态：`platform: 'wechat' | 'xiaohongshu'`

**验收标准**:
- [ ] 可以在两个平台间切换
- [ ] 切换后预览区正确更新
- [ ] 小红书模式显示尺寸选项

### F3: API 配置

**优先级**: P1（应该有）

**描述**: 用户配置自己的 AI API Key，用于 AI 分析和生图功能。

**交互规格**:
- 设置按钮（齿轮图标）打开配置弹窗
- 弹窗内容：
  - LLM 提供商选择（下拉）：千问 / 豆包 / DeepSeek / Kimi / GLM / GPT / Claude / Gemini / 自定义
  - API Key 输入框（密码模式，显示/隐藏切换）
  - API Base URL（自定义时填写）
  - 模型名称（自定义时填写）
  - 生图 API 配置（可选）：提供商选择 + Key
  - "测试连接"按钮
  - "保存"按钮
- API Key 保存在 `localStorage`，不传到任何服务器

**支持的 LLM 提供商**:

| 提供商 | API Base URL | 推荐模型 |
|-------|-------------|---------|
| 千问 (Qwen) | `https://dashscope.aliyuncs.com/compatible-mode/v1` | qwen-plus / qwen-turbo |
| 豆包 (Doubao) | `https://ark.cn-beijing.volces.com/api/v3` | doubao-1.5-pro |
| DeepSeek | `https://api.deepseek.com/v1` | deepseek-chat |
| Kimi (Moonshot) | `https://api.moonshot.cn/v1` | moonshot-v1-8k |
| GLM (智谱) | `https://open.bigmodel.cn/api/paas/v4` | glm-4-flash |
| OpenAI | `https://api.openai.com/v1` | gpt-4o-mini |
| Claude | `https://api.anthropic.com/v1` | claude-3-haiku |
| Gemini | `https://generativelanguage.googleapis.com/v1beta` | gemini-2.0-flash |
| 自定义 | 用户填写 | 用户填写 |

**支持的生图 API**:

| 提供商 | 用途 | 备注 |
|-------|------|------|
| 豆包 (Doubao) | 文生图 | 性价比高 |
| 千问 (Qwen) | 文生图 | 国内访问快 |
| Gemini | 文生图 | 原生图片生成 |
| OpenAI (DALL-E) | 文生图 | 质量高 |

**技术实现**:
- 统一的 OpenAI-compatible 接口格式（大部分国产模型都兼容）
- `localStorage` 存储配置
- 测试连接：发一个简单的 API 请求验证 Key 是否有效

**验收标准**:
- [ ] 可以选择不同的 LLM 提供商
- [ ] API Key 安全存储在 localStorage
- [ ] "测试连接"能正确验证
- [ ] 不填 API Key 也能使用核心排版功能

### F4: AI 分析

**优先级**: P1（应该有，但无 API Key 时可跳过）

**描述**: AI 分析用户的文章内容，识别文风和内容类型，推荐最匹配的排版方案。

**AI 需要判断的维度**:

| 维度 | 可能的值 | 影响什么 |
|------|---------|---------|
| 内容类型 | 知识干货 / 情感散文 / 新闻资讯 / 产品推广 / 生活随笔 / 访谈对话 | 排版结构 |
| 文风气质 | 专业严肃 / 温暖亲和 / 活泼年轻 / 文艺优雅 / 科技前沿 | 配色+装饰+字体 |
| 信息密度 | 高（干货密集） / 中 / 低（散文留白） | 排版结构+间距 |
| 色温倾向 | 暖色 / 冷色 / 中性 | 配色 |

**AI Prompt 模板**:

```
你是一个排版设计专家。请分析以下文章，返回 JSON 格式的排版建议。

文章内容：
---
{article_content}
---

请返回以下 JSON（不要添加任何其他文字）：
{
  "content_type": "knowledge|emotion|news|promotion|lifestyle|interview",
  "tone": "professional|warm|energetic|elegant|tech",
  "density": "high|medium|low",
  "color_temperature": "warm|cool|neutral",
  "recommended_color": "L1-L8 或 D1-D3 的编号",
  "recommended_layout": "T1-T5 的编号",
  "recommended_decoration": "S1-S4 的编号",
  "recommended_typography": "F1-F3 的编号",
  "reason": "一句话解释为什么推荐这个组合"
}
```

**技术实现**:
- 调用用户配置的 LLM API
- 解析 JSON 响应
- 如果解析失败，重试一次或回退到随机推荐
- 超时设置：10秒

**验收标准**:
- [ ] 能正确调用 AI API
- [ ] 能解析返回的 JSON
- [ ] 推荐结果能正确映射到原子组合
- [ ] API 失败时有优雅的降级（随机推荐）

### F5: 原子组合引擎

**优先级**: P0（必须）

**描述**: 根据选择的原子组合（配色+排版+装饰+字体），生成完整的排版方案。

**核心逻辑**:

```javascript
// 伪代码
function generateStyle(atoms) {
  const { colorScheme, layout, decoration, typography } = atoms;
  
  return {
    // 从配色方案获取具体色值
    colors: COLOR_SCHEMES[colorScheme],
    // 从排版结构获取 HTML 模板
    template: LAYOUT_TEMPLATES[layout],
    // 从装饰风格获取装饰元素
    decorations: DECORATION_SETS[decoration],
    // 从字体气质获取字体参数
    fonts: TYPOGRAPHY_SETS[typography],
  };
}
```

**"换一个"按钮逻辑**:
- 随机选择一个新的原子组合
- 但保持"不重复"——记录最近5次的组合，避免连续出现相同方案
- 动画过渡：旧方案淡出 → 新方案淡入

**验收标准**:
- [ ] 11×5×4×3 = 660种组合都能正确生成
- [ ] "换一个"不会连续出现相同方案
- [ ] 组合切换有平滑过渡动画

### F6: 微调面板

**优先级**: P1（应该有）

**描述**: 用户通过滑条对 AI 推荐的方案进行微调。

**面板设计**:

```
┌── 微调（默认折叠，点击展开）──────────┐
│                                      │
│  色温：暖色 ●─────────○ 冷色          │
│  明度：柔和 ○────●────○ 鲜明          │
│  装饰：极简 ───●────── 丰富           │
│  间距：紧凑 ─────●──── 宽松           │
│  圆角：硬朗 ○─────────● 柔和          │
│                                      │
│  [恢复 AI 推荐]                       │
└──────────────────────────────────────┘
```

**滑条映射逻辑**:

| 滑条 | 控制什么 | 值域 | 映射方式 |
|------|---------|------|---------|
| 色温 | 配色方案中的主色/辅色偏移 | 0-100 | 0=最暖，100=最冷 |
| 明度 | 背景色和文字色的对比度 | 0-100 | 0=柔和低对比，100=鲜明高对比 |
| 装饰 | 装饰元素的数量和复杂度 | 0-100 | 0=极简（只有线），100=丰富（SVG装饰） |
| 间距 | padding、margin、line-height | 0-100 | 0=紧凑，100=宽松 |
| 圆角 | border-radius 值 | 0-100 | 0=直角，100=大圆角 |

**技术实现**:
- HTML `<input type="range">` 滑条
- 滑条值变化时实时更新预览（debounce 100ms）
- "恢复 AI 推荐"按钮：重置所有滑条到 AI 推荐的位置

**验收标准**:
- [ ] 5根滑条都能正常拖动
- [ ] 拖动时预览实时更新
- [ ] "恢复AI推荐"能正确重置
- [ ] 面板默认折叠，点击展开

### F7: 实时预览

**优先级**: P0（必须）

**描述**: 实时展示排版效果。

**公众号模式预览**:
- 模拟微信文章的阅读视图
- 宽度固定为 375px（手机屏幕宽度）
- 背景色为微信的浅灰色
- 文章内容区白色背景

**小红书模式预览**:
- 显示图片组的缩略图列表
- 点击单张图片可放大查看
- 显示图片编号和总数（如 "1/6"）

**技术实现**:
- 使用 `<iframe>` 或 `<div>` 渲染预览
- 公众号模式：直接渲染 HTML+内联CSS
- 小红书模式：渲染每张图片的 HTML，用 html2canvas 转 PNG 预览

**验收标准**:
- [ ] 公众号预览宽度 375px
- [ ] 小红书预览显示图片组
- [ ] 修改设置后预览实时更新

### F8: 公众号导出

**优先级**: P0（必须）

**描述**: 将排版结果导出为微信公众号可用的格式。

**导出方式**:

| 方式 | 描述 | 用法 |
|------|------|------|
| **复制富文本** | 将渲染后的 HTML 复制到剪贴板 | 直接粘贴到微信公众号编辑器 |
| **导出HTML** | 下载完整的 HTML 文件 | 通过壹伴的"导入HTML"功能导入 |
| **复制源码** | 复制原始 HTML 代码 | 高级用户自行使用 |

**微信兼容性要求**（详见 04-TECH-ARCHITECTURE.md）:
- 所有样式使用内联 `style` 属性
- 不使用 `<style>` 标签
- 不使用 class/id 选择器
- 不使用 flex/grid 布局
- 使用 table 布局实现多列
- SVG 使用内联方式

**验收标准**:
- [ ] "复制富文本"后粘贴到微信编辑器，样式不丢失
- [ ] "导出HTML"后通过壹伴导入，样式正确
- [ ] 复制操作有成功提示

### F9: 小红书导出

**优先级**: P0（必须）

**描述**: 将内容生成为小红书图片组并下载。

**导出规格**:
- 格式：PNG（高质量）
- 分辨率：根据所选尺寸（默认 1080×1440 3:4）
- 图片命名：`yuntype-01.png`, `yuntype-02.png`, ...
- 打包下载：ZIP 格式一次性下载所有图片

**技术实现**:
- html2canvas 或 dom-to-image 库
- 渲染一个隐藏的 HTML 区域（目标尺寸）
- 截图生成 Canvas → 导出 PNG
- JSZip 库打包为 ZIP

**验收标准**:
- [ ] 生成的图片尺寸正确（1080×1440 等）
- [ ] 文字清晰不模糊
- [ ] 自定义字体正确渲染
- [ ] 可以单张下载或打包下载

### F10: AI 生图（配图）

**优先级**: P2（可以后做）

**描述**: 为文章生成配图，用户下载后自行插入。

**两种生图方式**:

| 方式 | 适用 | 技术 |
|------|------|------|
| **程序化信息图** | 知识类/科技类文章 | HTML+CSS 渲染 → 截图（无需AI） |
| **AI 文生图** | 情感类/故事类文章 | 调用生图API（豆包/千问/Gemini/DALL-E） |

**程序化信息图模板**:
- 流程图（步骤1→步骤2→步骤3）
- 对比表（A vs B）
- 知识卡片（标题+要点）
- 数据图表（柱状图/饼图，简单SVG）

**AI 文生图流程**:
```
用户标注故事点 → AI生成英文prompt → 调用生图API → 返回图片 → 用户下载
```

**验收标准**:
- [ ] 程序化信息图正确生成
- [ ] AI生图能调用API并返回结果
- [ ] 图片可以下载

## F11: 骨架系统

**优先级**: P0（V2 核心）

**描述**: 15+种布局骨架，决定文章的整体 HTML 结构（而非仅仅 CSS 参数变化）。

**骨架清单**:

| 编号 | 名称 | 结构特征 |
|------|------|---------|
| B01 | 经典简约 | 无装饰，干净排版，线条分割 |
| B02 | 色带章节 | 每个 h2 带彩色顶部色带 |
| B03 | 卡片模块 | 每章节包裹在圆角卡片中 |
| B04 | 左边栏重点 | 左侧竖色条贯穿章节 |
| B05 | 时间线 | 左侧时间线节点串联内容 |
| B06 | 杂志双栏 | 交替单栏/双栏布局 |
| B07 | 标签导航 | 顶部标签栏 + 分组内容 |
| B08 | 对话气泡 | Q&A 交替气泡 |
| B09 | 编号步骤 | 大编号 + 渐进步骤 |
| B10 | 引用高亮 | 大字引用穿插正文 |
| B11 | 网格画廊 | 图文网格交替 |
| B12 | 折叠手风琴 | 标题可展开/折叠（视觉模拟） |
| B13 | 渐变色块 | 渐变背景色块分区 |
| B14 | 侧注标注 | 右侧注释/旁批区域 |
| B15 | 报纸版式 | 多栏 + 大标题横跨 |

**交互规格**:
- 骨架选择面板：网格展示，每个骨架带缩略图
- 点击骨架即时切换，预览区立刻更新
- 骨架切换不影响已锁定的插槽选择

**验收标准**:
- [ ] 15种骨架全部能正确渲染
- [ ] 骨架切换<100ms 响应
- [ ] 骨架切换后自动匹配默认插槽搭配
- [ ] 微信公众号粘贴后样式不丢失

## F12: 插槽调节

**优先级**: P0（V2 核心）

**描述**: 10个维度的插槽可独立切换变体，实现精细控制。

**核心插槽（6个）**:

| 插槽 | 变体数 | 示例变体 |
|------|--------|---------|

| `quote` 引用 | 15+ | 左竖线、色块、气泡、便签纸、书页引用… |

## F13: 锁定机制

**优先级**: P1

**描述**: 用户锁定满意的插槽后，切换骨架或使用协调引擎时不被覆盖。

**交互规格**:
- 每个插槽旁有 🔒/🔓 切换按钮
- 锁定状态下：该插槽变体固定，骨架切换/协调引擎跳过此维度
- 解锁状态下：随骨架/协调引擎自动匹配
- 「全部解锁」快捷按钮

**验收标准**:
- [ ] 锁定插槽后切换骨架，该插槽保持不变
- [ ] 锁定状态持久化（刷新后保持）
- [ ] 「全部解锁」能一键重置

## F14: 调节模式

**优先级**: P1

**描述**: 两种模式控制插槽联动行为。

| 模式 | 行为 | 适合场景 |
|------|------|---------|
| **自由调节** | 每个插槽独立变化，互不影响 | 精确控制、有设计经验的用户 |
| **AI协调** | 修改一个插槽后，协调引擎自动调整其他未锁定插槽 | 快速获得和谐搭配 |

**验收标准**:
- [ ] 自由模式下修改标题不影响其他插槽
- [ ] AI协调模式下修改标题后，其他未锁定插槽自动匹配

---

## F15: 协调引擎

**优先级**: P1（本地规则引擎）/ P2（AI增强）

**描述**: 基于标签匹配和亲和关系，自动为未锁定插槽推荐匹配变体。

**本地规则引擎**:
- 每个变体有 `tags`（如 `minimal`、`modern`、`warm`）
- 每个变体有 `affinity` 映射（与其他变体的匹配度 0-1）
- 修改一个插槽后，根据 tags 交集和 affinity 得分排序，选最佳变体

**AI增强**（可选）:
- 调用 LLM 分析当前搭配，给出更智能的推荐
- 回退到本地规则引擎（无 API Key 时）

**验收标准**:
- [ ] 本地协调<50ms 响应
- [ ] 协调结果视觉和谐（主观测试通过）
- [ ] AI协调失败时自动回退本地规则
