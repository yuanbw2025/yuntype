> 写文章2小时，排版1小时——如果你也有同样的痛苦，这篇文章值得读到最后。

## 故事的起点

作为一个同时运营公众号和小红书的创作者，我每天的工作流是这样的：

打开Word写3000字 → 打开135编辑器排版（30分钟）→ 打开Canva做小红书配图（40分钟）→ 发现配色不统一，重来。

我试过秀米、135编辑器、Canva、甚至自己写CSS。没有一个工具能同时解决「公众号排版」和「小红书配图」两个需求。

于是我用 React + TypeScript，花了两周从零做了一个。

**它叫云中书 YunType**——取自李清照《一剪梅》"云中谁寄锦书来"。

完全免费，完全开源，MIT协议。没有会员，没有付费墙，没有"高级功能需要登录"。

[图1：云中书主界面全貌 — 三栏布局]

## 一、第一眼：三栏工作台

打开云中书，你会看到一个干净的三栏式工作台：

- **左栏**：Markdown编辑器，粘贴文章即可
- **中栏**：风格控制面板——骨架系统、配色、字体、插槽微调
- **右栏**：实时预览，所见即所得

顶栏有五个模式按钮：**V1经典**、**V2骨架**（公众号）、**小红书**、**信息图**，还有**AI分析**和**AI生图**。

不需要任何学习成本。粘贴文章 → 选个风格 → 复制到公众号。3步搞定。

## 二、核心武器：V2骨架+插槽系统

这是云中书最硬核的功能。

### 什么是骨架？

传统排版工具给你的是「模板」——一个固定的壳子，改不了。

云中书给你的是**骨架（Blueprint）**——它定义了文章的整体结构逻辑。目前有**15种骨架**：

| 骨架 | 风格 | 适合场景 |
|------|------|---------|
| 📝 极简清爽 | 无装饰，内容为王 | 技术教程、产品文档 |
| 🍵 日式留白 | 大量留白，禅意呼吸感 | 散文、读书笔记、生活随笔 |
| ✏️ 线条主导 | 线条分割，结构清晰 | 分析报告、知识科普 |
| 📚 双线学术 | 双线边框，学术严肃 | 论文笔记、学术分享 |
| 🏷️ 色块标签 | 彩色标签式标题 | 攻略指南、清单类内容 |
| 🌈 交替色带 | 奇偶段落交替底色 | 长文阅读、对比分析 |
| 📦 卡片模块 | 每节包裹在卡片中 | 产品对比、模块化内容 |
| 💬 气泡圆润 | 圆角气泡、柔和亲切 | 情感文章、用户访谈 |
| 📰 杂志编辑 | 大标题、首段放大 | 深度报道、品牌故事 |
| 🔤 首字下沉 | 段首大写字母下沉 | 文学作品、散文随笔 |
| 1️⃣ 编号步骤 | 自动编号标题 | 教程步骤、操作指南 |
| 📅 时间线 | 时间轴布局 | 项目进展、历史回顾 |
| ✒️ 文艺散文 | 居中标题、诗意排列 | 诗歌、散文、情感表达 |
| 💼 商务左标签 | 左侧色条标记 | 商业分析、会议纪要 |
| ◆ 几何装饰 | 几何符号点缀 | 设计类、创意类内容 |

点击骨架卡片，**整篇文章的结构一次性变化**。

[图3：15种骨架选择界面]

### 什么是插槽？

骨架决定了"大结构"，插槽决定"小细节"。每种骨架有**6个独立插槽**，你可以单独调整：

| 插槽 | 控制内容 | 变体举例 |
|------|---------|---------|
| 🔤 标题 | 标题的装饰方式 | 左对齐底线、居中对称、色块标签、双线框、Banner横幅、编号序号、日式细字、圆润气泡... |
| 💬 引用 | 引用块的视觉 | 左竖线、圆角卡片、居中大字引述、双线框、大引号、虚线框、气泡对话、摘要高亮框 |
| 📋 列表 | 列表的标记符号 | ● 圆点、■ 方块、▶ 箭头、🔹 菱形、①②③、— 短线 |
| ➖ 分割线 | 分割线的形态 | 细线、粗线、双线、渐变线、点状 · · ·、装饰花 ❖、菱形 ◇◇◇、居中 ○ |
| 📄 段落 | 段落的排列方式 | 无缩进紧凑、首行缩进、首字下沉、大留白舒展、首段放大、两端对齐 |
| 📦 节区 | 章节的包裹方式 | 平铺无包裹、卡片阴影、交替色带、时间线轨道、左侧标签、分隔线段落 |

**骨架是推荐搭配，插槽是精细调整。** 先选骨架"开箱好看"，再改插槽"个性定制"。

[图4：插槽微调下拉菜单]

### 排列组合有多少种？

```
15种骨架 × 10+标题变体 × 8+引用变体 × 8+列表变体 × 7+分割线变体 × 5+段落变体 × 5+节区变体
= 理论上超过 200,000 种独特组合
```

再加上11套配色 × 3种字体气质——你几乎不可能和别人"撞款"。

## 五、小红书模式：文章→图片组，一键生成

这是我自己用得最多的功能。

### 工作流

1. 粘贴Markdown文章
2. 点击顶栏「📸 小红书」
3. 选择比例：3:4（推荐）/ 1:1 / 16:9
4. 系统自动分页，生成图片组
5. 点击「打包下载 ZIP」

一篇3000字的文章 → 自动分成 40+ 张图 → 每张图配色统一、排版一致 → 打包下载。

从以前的1小时缩短到**3分钟**。

[图8：小红书模式 — 封面+缩略图+大图预览]

### 智能分页

系统解析Markdown为AST，估算每个元素的高度，按画布尺寸自动切分。第一页永远是封面，最后一页是尾页。

### 五种封面变体

每次生成，封面从5种变体中随机选择：经典居中、大字冲击、极简留白、卡片包裹、杂志风。

### 风格联动

小红书的配色、字体跟公众号共享同一套系统——你的小红书配图和公众号文章**视觉完全统一**。

[图9：小红书四维度面板]

## 七、AI加持

### AI智能分析

点击顶栏的 **🤖 AI分析** 按钮。没有API Key也能用——自动降级为**本地关键词分析**（基于规则引擎）。

配了API Key则调用大模型深度分析文章类型、气质、受众，推荐最佳排版组合。

支持**9大AI提供商**：通义千问、DeepSeek、豆包、OpenAI、Gemini、Moonshot(Kimi)、智谱(GLM)、SiliconFlow、自定义。

**隐私设计**：API Key直连提供商，数据不经过任何中间服务器。

## 八、纯文本一键整理

最新加入的功能。

很多人写文章不用Markdown，而是直接写纯文本。粘贴到云中书后，点击左上角的 **🪄 整理** 按钮：

- 自动识别标题（独立短行、中文编号"一、二、三"）
- 自动识别列表（数字/符号开头）
- 自动识别引用（引号包裹的段落）
- 自动段落分隔

一键把乱糟糟的纯文本变成结构清晰的Markdown。

## 十、技术极客看这里

| 技术 | 选择 |
|------|------|
| 框架 | React 19 + TypeScript 5.7 |
| 构建 | Vite 6 |
| 样式 | **100% 内联CSS**（微信兼容的关键） |
| Markdown | **自研解析器**（不依赖marked/remark） |
| 图片导出 | html2canvas + JSZip（懒加载） |
| 离线 | Service Worker / PWA |
| UI库 | **无**（零UI组件库依赖） |

### 为什么用纯内联CSS？

微信公众号编辑器不支持 `<style>` 标签、不支持CSS class，只支持 `style="..."` 内联样式。

所以云中书生成的HTML**全部使用内联style属性**。复制到公众号后排版100%还原。

### 为什么自研Markdown解析器？

精准控制AST + 体积极小（200行代码）+ 一次遍历完成 + 直接支持小红书分页算法。

### 首屏仅95KB

核心代码95KB gzip，html2canvas等重量级库通过 `dynamic import()` 按需加载。
